# Таблица уровней стажёров LM.inc

Статический сайт с таблицей рейтинга (место / номер пропуска / баллы / уровень),
которая автоматически обновляется данными с
`https://lm-inc-levels-knyazhna.amvera.io`.

## Как это работает

- `index.html` — сама страница с таблицей. Читает `data.json` из этого же
  репозитория (без CORS-проблем, т.к. файл лежит рядом) и рендерит таблицу,
  отсортированную по баллам.
- `data.json` — снимок данных (сотрудники, серийные номера, баллы, уровень).
- `scripts/update-data.mjs` — Node-скрипт, который скачивает страницу-источник,
  вытаскивает из неё встроенный список сотрудников и пересобирает `data.json`.
- `.github/workflows/update-data.yml` — GitHub Action, который запускает этот
  скрипт по расписанию (каждые 15 минут) и коммитит `data.json`, если данные
  изменились.

Таким образом таблица обновляется сама, без вашего участия: раз в 15 минут
Action проверяет источник → если баллы/уровни изменились → обновляет
`data.json` в репозитории → сайт на GitHub Pages подхватывает новый файл.

## Как выложить на GitHub Pages

1. Создайте новый репозиторий на GitHub (например, `lm-inc-levels`).
2. Загрузите в него все файлы из этой папки, сохранив структуру:
   ```
   index.html
   data.json
   README.md
   scripts/update-data.mjs
   .github/workflows/update-data.yml
   ```
   Проще всего — через `git`:
   ```bash
   cd lm-inc-levels
   git init
   git add .
   git commit -m "Первый коммит"
   git branch -M main
   git remote add origin https://github.com/ВАШ_ЛОГИН/lm-inc-levels.git
   git push -u origin main
   ```
3. В репозитории откройте **Settings → Pages**.
4. В разделе **Build and deployment** выберите:
   - Source: **Deploy from a branch**
   - Branch: **main**, папка **/ (root)**
5. Сохраните — через минуту-две сайт будет доступен по адресу вида
   `https://ВАШ_ЛОГИН.github.io/lm-inc-levels/`.
6. Проверьте вкладку **Actions** — там должен появиться workflow
   «Обновление данных LM.inc». Он выполнится автоматически по расписанию,
   но можно сразу запустить вручную кнопкой **Run workflow**, чтобы обновить
   данные сразу после публикации.

## Важно про права Action на пуш в репозиторий

Если Action не сможет запушить обновлённый `data.json` (ошибка permissions),
зайдите в **Settings → Actions → General → Workflow permissions** и
выберите **Read and write permissions**, затем сохраните.

## Локальная проверка

```bash
node scripts/update-data.mjs   # обновит data.json из реального источника
python3 -m http.server 8000    # или любой другой статический сервер
# откройте http://localhost:8000
```

## Регистрация через Google Таблицу (`register/`)

Страница `register/index.html` — форма «Имя / Фамилия / Номер пропуска».
После заполнения:

1. Введённый номер пропуска сверяется с `data.json` (чтобы убедиться, что
   пропуск существует).
2. Данные отправляются в Google Таблицу (через Google Apps Script — см. ниже).
3. Сразу после этого показывается та же таблица рейтинга, что и на главной
   странице, с подсветкой строки зарегистрировавшегося.

Открывается она по адресу вида
`https://ВАШ_ЛОГИН.github.io/ИМЯ_РЕПОЗИТОРИЯ/register/`.

### Почему через Google Таблицу, а не напрямую в GitHub

Раньше эта форма писала прямо в файл в репозитории через GitHub-токен. Так
делать нельзя: GitHub автоматически находит и отзывает любые свои токены,
случайно опубликованные в открытом (публичном) репозитории — это встроенная
защита (secret scanning), обойти её нельзя, только спрятав токен на
отдельном сервере. Поэтому для сайта без сервера правильный вариант —
использовать сервис, который не требует секретного ключа со стороны браузера.
Google Таблица + Apps Script отлично для этого подходит: сама ссылка на
веб-приложение не секретна, её не нужно прятать.

### Настройка (один раз)

1. Создайте Google Таблицу на sheets.google.com.
2. В ней: **Расширения → Apps Script**. Вставьте туда код из файла
   `google-apps-script.js` (он в этом же архиве), полностью заменив
   код-заглушку. Сохраните.
3. **Deploy → New deployment → Web app**:
   - Execute as: **Me**
   - Who has access: **Anyone**
4. Нажмите **Deploy**, скопируйте появившуюся ссылку
   (`https://script.google.com/macros/s/.../exec`).
5. Откройте `register/config.js`, вставьте эту ссылку в поле
   `GAS_WEB_APP_URL`.
6. В самой таблице: **Файл → Поделиться → Опубликовать в интернете**,
   выберите лист и формат **CSV**, нажмите «Опубликовать», скопируйте ссылку.
7. Откройте `admin/index.html`, вставьте эту ссылку в константу
   `REGISTRATIONS_CSV_URL` в начале `<script>`.

Никакого токена нигде хранить больше не нужно.

## Страница для организаторов (`admin/`)

Страница `admin/index.html` показывает список всех, кто уже
зарегистрировался (Фамилия Имя, номер пропуска, дата и время) — читает
опубликованную Google Таблицу напрямую (см. настройку выше), ничего никуда
не пишет.

Доступ по паролю **445674** (задан прямо в коде страницы, в переменной
`PASSWORD` внутри `admin/index.html` — если хотите сменить пароль, поменяйте
там). Это не «настоящая» защита (пароль виден любому, кто откроет исходный
код страницы), а простая преграда от случайных посетителей — этого более
чем достаточно, если в списке нет ничего секретного (как в случае с лагерем:
только имена, фамилии и номера пропусков).

Открывается по адресу
`https://ВАШ_ЛОГИН.github.io/ИМЯ_РЕПОЗИТОРИЯ/admin/`.
Ссылка на неё также есть внизу главной страницы и страницы регистрации.

## Если сайт-источник изменит вёрстку

Скрипт `scripts/update-data.mjs` ищет в HTML-странице подстроку
`"employees":[...]` (это встроенные Next.js данные). Если разработчики
источника изменят структуру страницы и поиск перестанет находить эти данные,
Action просто завершится с ошибкой в логах (`data.json` при этом не
перезатирается — сайт продолжит показывать последние актуальные данные),
и вам нужно будет посмотреть свежий HTML страницы и поправить скрипт.
